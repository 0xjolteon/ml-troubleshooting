WARP with Split Tunneling

The standard WARP setup routes all your internet traffic through Cloudflare. This guide sets up Cloudflare Zero Trust with split tunneling so that only traffic goes through WARP — everything else uses your normal connection.

Why bother? Sending all traffic through WARP can slow down other browsing and streaming. Split tunneling gives you the routing benefit for the game server while leaving the rest of your internet untouched.
What you'll need: a free Cloudflare account (sign up at cloudflare.com), about 10–15 minutes, and a Windows PC. The Cloudflare One Client is also available for macOS and Linux — the dashboard steps are identical across platforms.
1
Install the Cloudflare One Client
  1. Download the Cloudflare One Client (this is the same app as WARP) from one.one.one.one or directly from Cloudflare's download page.
    Windows link: downloads.cloudflareclient.com/v1/download/windows/ga
  2. Run the installer and follow the prompts through to completion.
  3. When the app opens for the first time, it will ask "What would you like to use WARP for?" — choose Cloudflare One Client (the right option). Do not choose "Private browsing" — that mode doesn't support split tunneling.

After selecting Cloudflare One Client, the app will show a disconnected state. Leave it here for now — you'll come back to connect it after configuring the dashboard.

2
Set up Cloudflare Zero Trust
  1. Go to cloudflare.com and log in to your account (or create a free one). In the left sidebar, find Protect & Connect → Zero Trust.
  1. The first time you open Zero Trust, you'll be asked to set a team name. This is a unique identifier for your Zero Trust organization — pick something memorable (e.g. your server name). This can't be easily changed later, so choose carefully.
3
Configure Device Enrollment

Before anyone can enroll their device into your Zero Trust organization, you need to create an enrollment policy that defines who is allowed. For a private server, the simplest approach is to allow enrollment by email — each player enrolls with their own email address.

  1. In the Zero Trust dashboard, go to Team & Resources → Devices in the left sidebar, then click the Management tab at the top.
  1. Under Device enrollment, click Manage. Then click Create a new policy.
  2. Configure the policy:
    • Set Policy Name to something descriptive (e.g. "Allow email enrollment")
    • Set Action to Allow
    • Under Policy rules, set the selector to Emails and enter your email address
    • Leave session duration as default
  1. Click Save on the policy, then click Save again on the overall enrollment settings page. Both saves are required — the policy won't take effect if you only save the inner form.
You must click Save twice — once for the policy itself, and once for the enrollment settings page. Skipping the second save means the policy doesn't apply and enrollment will fail.
4
Enroll Your Device

Now go back to the Cloudflare One Client app you installed earlier.

  1. In the app, click Settings → Preferences → Account (or look for "Login with Cloudflare Zero Trust"). You'll be prompted to enter your team name.
  2. Enter the team name you set up in Step 2 (just the name, not the full domain — e.g. myservername not myservername.cloudflareaccess.com), then click Continue.
  1. A browser window will open showing a Cloudflare Access login page asking for your email. Enter the same email address you put in the enrollment policy.
  1. Cloudflare will send a 6-digit one-time code to that email. Enter it in the browser to verify your identity.
  2. The browser will prompt you to open the Cloudflare One Client — allow it. The app will show a waiting dialog while it receives confirmation.

Once authenticated, the app will show your enrollment is complete. Your device is now enrolled in your Zero Trust organization.

5
Configure Split Tunneling

This is the key step. By default, Zero Trust routes all traffic through Cloudflare. You'll change it to Include mode — which means only the domains and IPs you specify will go through WARP. Everything else (streaming, browsing, Discord) uses your normal connection.

  1. In the Zero Trust dashboard, go to Team & Resources → Devices → Device Profiles. Click the ⋯ (three dots) next to the Default profile and select Configure.
  1. Scroll down to the Split Tunnels section. You'll see it is set to Exclude IPs and domains by default — this sends everything through WARP except what's listed. You need to change this.
  2. Change the mode to Include IPs and domains. This flips the logic — now only traffic to destinations you add will go through WARP. Click Manage to open the include list.
  1. You'll now add the server's domains to the include list. For each entry, click Add a destination, set the selector to Domain, and add the following:
    • — the main server site
    • *.legends.ml — wildcard to cover all subdomains
    • The game server's direct hostname or IP (if different from the website)
    • Any image hosts or CDN domains the forum uses
    Give each entry a meaningful description so the list stays readable.

6
Connect and Test
  1. Open the Cloudflare One Client and click Connect. The toggle turns blue when active.
  2. Try loading the server site. Because only your specified domains are tunneled through WARP, you should now be able to reach it while the rest of your internet works exactly as before.
  3. To verify split tunneling is working correctly, load a regular site like cloudflare.com/cdn-cgi/trace — it should show warp=off, confirming that general traffic is not going through WARP. Then check a server-specific URL — that traffic will be tunneled.
You're done! WARP is now active only for traffic. You won't notice any difference for everything else — streaming, Discord, and other games all continue using your regular connection.

Sharing this setup with other players

Each player needs to create their own free Cloudflare account and repeat this process. However, you only need to create the Zero Trust organization once — other players can enroll into your organization instead of making their own, as long as you add their email to the enrollment policy.

To add more players: go to Devices → Management → Device enrollment → Manage, open your policy, and add their email address under the Emails rule. They can then enroll using your team name and their own email.

The free Cloudflare Zero Trust plan supports up to 50 users — more than enough for a private server community.

Troubleshooting

Enrollment fails or says "team not found"

Double-check you're entering just the team name (e.g. myserver), not the full domain (myserver.cloudflareaccess.com).

Email code never arrives

Check your spam folder. Codes expire after a few minutes — if it expired, restart the enrollment flow.

Site still blocked after connecting WARP

Make sure you added the correct domain to the split tunnel include list and saved. Also try disconnecting and reconnecting WARP after making changes to the dashboard — the client may cache the old config for a few minutes.

All my internet went through WARP (split tunnel not working)

Confirm the Split Tunnel mode is set to Include (not Exclude). The default is Exclude — if you didn't change it, all traffic routes through WARP.
