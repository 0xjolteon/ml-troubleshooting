Antivirus

Whitelist the site in your antivirus

Some antivirus products block .ml domains at the application level, separate from DNS. If changing DNS didn't help, your antivirus web shield may be the cause.

How to tell if antivirus is the problem

If your browser shows a warning page from your antivirus (not just a generic browser error), that's a clear sign. Look for phrases like "Web Shield blocked this page" or the antivirus name in the warning.

You can also try loading the site in a private/incognito window — antivirus web shields usually still apply, so if it's still blocked there, DNS is probably not the issue.

Whitelist by antivirus Pick yours

  1. Open Avast (or AVG) and go to Menu → Settings.
  2. Go to General → Exceptions.
  3. Click Add Exception.
  4. Enter the full URL of the server site and click Add Exception to confirm.
AVG's interface is nearly identical to Avast — follow the same steps.
  1. Open Malwarebytes and click Settings (gear icon).
  2. Go to the Allow List tab.
  3. Click Add and select Allow a Website.
  4. Enter the server domain and click Confirm.
If you're using Malwarebytes Browser Guard (the extension), you'll also need to click the extension icon in your browser, find the blocked entry, and click Trust this website.

Windows Defender's SmartScreen is the most likely culprit here. It may show an "Unsafe website" warning that you can bypass.

  1. When you see the SmartScreen block page, click More information (or the small text at the bottom).
  2. Click Go to the site anyway (unsafe). This creates a one-time exception.

To permanently whitelist in Windows Security:

  1. Open Windows Security → App & browser control → Reputation-based protection settings.
  2. SmartScreen for Microsoft Edge can be toggled here, but per-site whitelisting is limited. The most reliable fix is switching DNS as described in the DNS guide.
  1. Open Kaspersky and go to Settings → General → Exclusions and actions on object detection.
  2. Click Manage Exclusions.
  3. Click Add and enter the server URL.
  4. Make sure to check the components you want to exclude — at minimum, select Web Anti-Virus.
  5. Click Add to save.
  1. Open Bitdefender and go to Protection → Online Threat Prevention.
  2. Click Settings under Online Threat Prevention.
  3. Scroll to Whitelist and click Manage Whitelist.
  4. Click Add, enter the server URL, and click Add to confirm.

Browser extension blockers

If your antivirus installs a browser extension (many do), that extension may block the site independently. Check your extensions (usually at chrome://extensions or about:addons) for anything from your antivirus vendor and look for a whitelist or allow-list option within it.

Common culprits: Avast Online Security, AVG Online Security, Kaspersky Protection, Bitdefender TrafficLight.

Still blocked?

If whitelisting didn't help, the block is likely at the DNS level from your ISP rather than from your antivirus. Head to the DNS fix guide or run the troubleshooter to diagnose further.
