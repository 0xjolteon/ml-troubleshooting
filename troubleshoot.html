🪟 Windows — Step by step

Follow this chart if you're on Windows and the site won't load.

flowchart TD
    A([Start — Site won't load on Windows]) --> B{Does a browser\nerror name your\nantivirus?}
    B -->|Yes| AV[Open Avast / Malwarebytes\n/ Defender and whitelist\nthe site URL]
    B -->|No| C[Open Command Prompt\nRun: nslookup yourserver.ml]
    C --> D{Did it return\nan IP address?}
    D -->|No — DNS blocked| E[Change DNS to 1.1.1.1\nSettings → Network\n→ DNS → Manual]
    D -->|Yes — DNS works| AV
    E --> F[Run: ipconfig /flushdns\nthen reload site]
    F --> G{Site loads now?}
    G -->|Yes ✅| DONE([✅ Done!])
    G -->|No| AV
    AV --> H{Site loads now?}
    H -->|Yes ✅| DONE
    H -->|No| WARP[Install Cloudflare WARP\nfrom one.one.one.one\nand toggle it on]
    WARP --> I{Site loads now?}
    I -->|Yes ✅| DONE
    I -->|No| HELP([💬 Post in Discord with:\nnslookup result + OS version\n+ antivirus name])

    style A fill:#f7931e,color:#000,stroke:none

Tip: Run nslookup yourserver.ml in Command Prompt first. If it returns can't find server, DNS is the problem. If it returns an IP, antivirus is the problem.

🍎 macOS — Step by step

Follow this chart if you're on macOS and the site won't load.

flowchart TD
    A([Start — Site won't load on macOS]) --> B[Open Terminal\nRun: dig yourserver.ml]
    B --> C{Did dig return\nan IP address?}
    C -->|No — DNS blocked| D[System Settings → Network\n→ your connection\n→ Details → DNS tab\nAdd 1.1.1.1 and 1.0.0.1]
    C -->|Yes — DNS works| AV
    D --> E[Flush cache:\nsudo dscacheutil -flushcache\nsudo killall -HUP mDNSResponder]
    E --> F{Site loads now?}
    F -->|Yes ✅| DONE([✅ Done!])
    F -->|No| AV[Check for antivirus\nbrowser extensions\nor security software\nand whitelist the site]
    AV --> G{Site loads now?}
    G -->|Yes ✅| DONE
    G -->|No| WARP[Install Cloudflare WARP\nfrom the Mac App Store\nor one.one.one.one]
    WARP --> H[Toggle WARP on\nVerify at:\ncloudflare.com/cdn-cgi/trace\nCheck warp=on]
    H --> I{Site loads now?}
    I -->|Yes ✅| DONE
    I -->|No| HELP([💬 Post in Discord with:\ndig output + macOS version\n+ security software installed])

    style A fill:#f7931e,color:#000,stroke:none

Tip: Run dig yourserver.ml in Terminal. Look for an ;; ANSWER SECTION: with an IP address. No answer section means DNS is blocked.

🐧 Linux — Step by step

Follow this chart if you're on Linux and the site won't load.

flowchart TD
    A([Start — Site won't load on Linux]) --> B[Run: dig yourserver.ml\nor: nslookup yourserver.ml]
    B --> C{Did it return\nan IP address?}
    C -->|No — DNS blocked| D{Which DNS\nmanager are you using?}
    C -->|Yes — DNS works| BROWSER[Check browser\nsecurity extensions\nand whitelist site]
    D --> SD[systemd-resolved:\nEdit /etc/systemd/resolved.conf\nSet DNS=1.1.1.1 1.0.0.1\nRestart: sudo systemctl restart\nsystemd-resolved]
    D --> NM[NetworkManager:\nnmcli con mod connection-name\nipv4.dns 1.1.1.1\nnmcli con up connection-name]
    D --> RC[Static resolv.conf:\nEdit /etc/resolv.conf\nnameserver 1.1.1.1\nnameserver 1.0.0.1]
    SD --> FLUSH[Flush cache:\nsudo systemd-resolve\n--flush-caches]
    NM --> TEST
    RC --> TEST
    FLUSH --> TEST{Site loads now?}
    TEST -->|Yes ✅| DONE([✅ Done!])
    TEST -->|No| BROWSER
    BROWSER --> G{Site loads now?}
    G -->|Yes ✅| DONE
    G -->|No| WARP[Install Cloudflare WARP\nSee WARP guide for\napt/yum install steps]
    WARP --> H[warp-cli registration new\nwarp-cli connect\nwarp-cli status]
    H --> I{Site loads now?}
    I -->|Yes ✅| DONE
    I -->|No| HELP([💬 Post in Discord with:\ndig output + distro name\n+ DNS manager used])

    style A fill:#f7931e,color:#000,stroke:none

Not sure which DNS manager you have? Run systemctl is-active systemd-resolved and systemctl is-active NetworkManager — whichever returns active is your manager.
