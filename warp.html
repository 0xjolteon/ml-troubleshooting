Set up Cloudflare WARP

WARP is a free tool from Cloudflare that improves your internet routing. If you're experiencing frequent disconnections or can't reach the server at all, WARP often fixes both problems — no VPN subscription needed.

What is WARP and how does it help?

WARP is not a traditional VPN. It doesn't change your IP address or hide your location. Instead, it routes your traffic through Cloudflare's global network, which often takes a much more direct path than your ISP's default routing — reducing latency and packet loss for players whose ISP routes traffic inefficiently to our server region.

Two problems, one fix: Because WARP uses Cloudflare's own DNS (1.1.1.1), it automatically bypasses ISP-level domain blocking. If you're having trouble reaching and experiencing disconnects, try WARP first before anything else.
WARP is completely free. There is a paid tier (WARP+) with faster speeds, but the free version is sufficient for gaming and fixing the issues described here.

Install WARP Pick your OS

  1. Download the installer from one.one.one.one — click the Windows download button.
  2. Run the downloaded .exe installer — just click through the defaults.
  3. Once installed, WARP appears in your system tray (bottom-right). Click the icon to open it.
  4. Click the large toggle to connect. The button turns blue and shows Connected.
  5. Try loading again.
If WARP was blocked by antivirus during install, see the antivirus guide to whitelist it first.
  1. Download the macOS client from one.one.one.one or the Mac App Store (search "1.1.1.1").
  2. Open the downloaded .pkg file and follow the installer. Click Allow when prompted for VPN configuration — this is normal.
  3. WARP appears in your menu bar (top-right). Click the icon.
  4. Click the toggle to connect. It will turn blue when connected.
  5. Try loading again.
macOS may ask for your password when installing the VPN profile. This is expected — WARP uses a VPN tunnel interface even though it's not a traditional VPN.

Debian / Ubuntu

curl -fsSL https://pkg.cloudflareclient.com/pubkey.gpg | \
  sudo gpg --yes --dearmor -o /usr/share/keyrings/cloudflare-warp-archive-keyring.gpg

echo "deb [arch=amd64 signed-by=/usr/share/keyrings/cloudflare-warp-archive-keyring.gpg] \
  https://pkg.cloudflareclient.com/ $(lsb_release -cs) main" | \
  sudo tee /etc/apt/sources.list.d/cloudflare-client.list

sudo apt update && sudo apt install cloudflare-warp

Fedora / RHEL / CentOS

curl -fsSl https://pkg.cloudflareclient.com/cloudflare-warp-ascii.repo | \
  sudo tee /etc/yum.repos.d/cloudflare-warp.repo
sudo yum install cloudflare-warp

Connect after installing (all distros)

warp-cli registration new
warp-cli connect
warp-cli status
You should see Status update: Connected.

Disconnect when not needed

warp-cli disconnect

Confirm WARP is working

Visit cloudflare.com/cdn-cgi/trace in your browser and look for warp=on.

Still disconnecting after WARP?

  • In the app settings, switch to full WARP mode rather than DNS-only mode — it routes more traffic and typically helps more with gaming stability.
  • Check if disconnects only happen at specific times — peak-hour ISP congestion can't always be fully overcome by WARP.
  • Run the troubleshooter for a detailed diagnosis, or ask in our Discord in .
  • For a more advanced setup that only routes server traffic through WARP, see the Zero Trust split tunneling guide.
