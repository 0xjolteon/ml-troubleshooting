Set up Cloudflare WARP

WARP is a free tool from Cloudflare that improves your internet routing. For SEA players, it often fixes frequent disconnections and the .ml DNS block — both at once, no VPN subscription needed.

What is WARP and how does it help?

WARP is not a traditional VPN. It doesn't change your IP address or hide your location. Instead, it routes your traffic through Cloudflare's global network, which often takes a much more direct path than your ISP's routing — especially from Southeast Asia to servers hosted in other regions.

Two problems, one fix: Because WARP uses Cloudflare's own DNS (1.1.1.1), it automatically bypasses ISP-level .ml domain blocking. If you're a SEA player having trouble reaching the site, try WARP first before anything else.
WARP is completely free. There is a paid tier (WARP+) with faster speeds, but the free version is sufficient for gaming and fixing the connection issues described here.

Install WARP Pick your OS

  1. Download the installer from one.one.one.one — click the Windows download button. (If that page won't load, search "Cloudflare WARP download" or try the direct link: developers.cloudflare.com/warp-client/get-started)
  2. Run the downloaded .exe installer. It requires no special configuration — just click through the defaults.
  3. Once installed, WARP appears in your system tray (bottom-right). Click the icon to open it.
  4. Click the large toggle to connect. The button turns blue and shows Connected when it's working.
  5. Try loading the server site again.
If WARP was blocked by antivirus during install, see the antivirus guide to whitelist it first.

macOS may ask for your password when installing the VPN profile. This is expected — WARP uses a VPN tunnel interface even though it's not a traditional VPN.

WARP on Linux is installed via Cloudflare's package repository. Choose your distro:

Debian / Ubuntu

# Add Cloudflare's GPG key and repo
curl -fsSL https://pkg.cloudflareclient.com/pubkey.gpg | \
  sudo gpg --yes --dearmor -o /usr/share/keyrings/cloudflare-warp-archive-keyring.gpg

echo "deb [arch=amd64 signed-by=/usr/share/keyrings/cloudflare-warp-archive-keyring.gpg] \
  https://pkg.cloudflareclient.com/ $(lsb_release -cs) main" | \
  sudo tee /etc/apt/sources.list.d/cloudflare-client.list

sudo apt update && sudo apt install cloudflare-warp

Fedora / RHEL / CentOS

curl -fsSl https://pkg.cloudflareclient.com/cloudflare-warp-ascii.repo | \
  sudo tee /etc/yum.repos.d/cloudflare-warp.repo

sudo yum install cloudflare-warp

Connect after installing (all distros)

# Register the client (first time only)
warp-cli registration new

# Connect
warp-cli connect

# Check status
warp-cli status
You should see Status update: Connected. If you see an error, run warp-cli disconnect then warp-cli connect again.

Disconnect when not needed

warp-cli disconnect

Confirm WARP is working

Visit cloudflare.com/cdn-cgi/trace in your browser. Look for the line that says warp=on. If it shows warp=off, the toggle may not be connected — open the WARP app and try toggling it off and back on.

Still disconnecting after WARP?

WARP helps with routing issues caused by your ISP's path to our server, but it won't fix everything. If you're still experiencing disconnects:

  • Try switching WARP's mode — in the app settings, look for WARP mode vs DNS-only mode. Full WARP mode routes more traffic and typically helps more with gaming stability.
  • Check if the issue only happens at specific times of day — peak hours can cause congestion that WARP can't fully overcome.
  • Run the troubleshooter for a more detailed diagnosis.
