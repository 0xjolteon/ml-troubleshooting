DNS Fix

Fix the DNS block

Your ISP's DNS resolver may be blocking our domain. Switching to a public DNS like Cloudflare or Google bypasses this completely - it takes about 2 minutes.

What is DNS and why does this work?

When you type a web address, your computer asks a DNS resolver to translate it to an IP address. By default this is your ISP's resolver - and your ISP may have flagged domains as suspicious. Switching to a public DNS resolver (like 1.1.1.1 from Cloudflare or 8.8.8.8 from Google) bypasses your ISP's blocklist entirely.

This does not change your internet speed or affect anything else - it only changes how domain names get looked up.

Change your DNS Pick your OS

Option A - Settings (Windows 10 / 11)

  1. Press Win + I to open Settings, then go to Network & Internet.
  2. Click your active connection (Wi-Fi or Ethernet), then click Edit next to DNS server assignment.
  3. Change the dropdown from Automatic (DHCP) to Manual.
  4. Toggle on IPv4 and enter: 
    Preferred DNS:  1.1.1.1
Alternate DNS:  1.0.0.1
  5. Click Save and reload the site.

Option B - Command Prompt (faster)

Open Command Prompt as Administrator and run:

netsh interface ip set dns name="Wi-Fi" static 1.1.1.1
netsh interface ip add dns name="Wi-Fi" 1.0.0.1 index=2
Replace Wi-Fi with Ethernet if you're on a wired connection. Run netsh interface show interface to see your adapter names.

Option C - Flush DNS cache after changing

ipconfig /flushdns

System Settings (macOS Ventura / Sonoma)

  1. Open System Settings → Network.
  2. Click your active connection (Wi-Fi or Ethernet), then click Details…
  3. Click the DNS tab.
  4. Click the + button and add: 
    1.1.1.1
1.0.0.1
  5. Click OK and then Apply.

Terminal method

sudo networksetup -setdnsservers Wi-Fi 1.1.1.1 1.0.0.1
Replace Wi-Fi with Ethernet if wired. List your services with networksetup -listallnetworkservices.

Flush DNS cache after changing

sudo dscacheutil -flushcache; sudo killall -HUP mDNSResponder
Linux DNS configuration varies by distro and whether you use systemd-resolved, NetworkManager, or a static resolv.conf. Check which one applies to you first.

Check which system you have

systemctl is-active systemd-resolved
systemctl is-active NetworkManager

systemd-resolved (Ubuntu 20.04+, Fedora, Arch)

  1. Edit the config file: 
    sudo nano /etc/systemd/resolved.conf
  2. Find or add the [Resolve] section and set: 
    [Resolve]
DNS=1.1.1.1 1.0.0.1
FallbackDNS=8.8.8.8 8.8.4.4
  3. Restart the service: 
    sudo systemctl restart systemd-resolved

NetworkManager (most desktop distros)

nmcli con mod "$(nmcli -t -f NAME con show --active | head -1)" \
  ipv4.dns "1.1.1.1 1.0.0.1" \
  ipv4.ignore-auto-dns yes
nmcli con up "$(nmcli -t -f NAME con show --active | head -1)"

Static /etc/resolv.conf (fallback)

Only use this if neither method above applies. On modern systems, resolv.conf is managed automatically and your edits may be overwritten on reboot.
sudo nano /etc/resolv.conf

Replace the contents with:

nameserver 1.1.1.1
nameserver 1.0.0.1

Flush DNS cache

# systemd-resolved
sudo systemd-resolve --flush-caches
# nscd (if installed)
sudo systemctl restart nscd

Verify it worked

After changing your DNS, run this to confirm you're using Cloudflare's resolver:

# Windows (Command Prompt)
nslookup one.one.one.one

# macOS / Linux
dig +short whoami.cloudflare.com TXT @1.1.1.1

Then try loading again - if it still doesn't work, check the antivirus guide or run the troubleshooter.
